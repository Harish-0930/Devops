# Terraform Reference Guide - Expressions, Functions,  Backend Types and CLI Commands


## Expressions & Functions

### Local Values

Locals allow you to define reusable, computed values within a module.

```hcl
locals {
  common_tags = {
    Project     = "TerraformGuide"
    ManagedBy   = "Terraform"
    Environment = var.environment
  }

  # Computed local value
  full_bucket_name = "${var.project}-${var.environment}-data"
}

resource "aws_s3_bucket" "data" {
  bucket = local.full_bucket_name
  tags   = local.common_tags
}
```

---

### Conditional Expressions

Syntax: `condition ? true_value : false_value`

```hcl
resource "aws_instance" "server" {
  instance_type = var.environment == "prod" ? "t3.large" : "t2.micro"
  ami           = "ami-0c55b159cbfafe1f0"
}
```

---

### `count` Meta-Argument

Use `count` to create multiple identical resources.

```hcl
# Create 3 identical EC2 instances
resource "aws_instance" "web" {
  count         = 3
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  tags = {
    Name = "web-${count.index}"  # web-0, web-1, web-2
  }
}
```

---

### `for_each` Meta-Argument

Preferred over `count` for named resources — iterates over a map or set.

```hcl
variable "buckets" {
  default = {
    logs    = "us-east-1"
    backups = "us-west-2"
    archive = "eu-west-1"
  }
}

resource "aws_s3_bucket" "buckets" {
  for_each = var.buckets
  bucket   = "myapp-${each.key}"

  tags = {
    Region = each.value
  }
}
```

---

### `for` Expressions

Transform lists and maps inline.

```hcl
variable "names" { default = ["alice", "bob", "carol"] }

locals {
  upper_names = [for name in var.names : upper(name)]
  # Result: ["ALICE", "BOB", "CAROL"]

  name_lengths = { for name in var.names : name => length(name) }
  # Result: { alice = 5, bob = 3, carol = 5 }
}
```

### Built-in Functions

#### String Functions

```hcl
length("hello")                      # 5
upper("hello")                       # "HELLO"
lower("HELLO")                       # "hello"
trimspace("  hi  ")                  # "hi"
format("%-10s", "hello")             # "hello     "
replace("foo-bar", "-", "_")         # "foo_bar"
```

#### Collection Functions

```hcl
length(["a","b","c"])                # 3
concat(["a"], ["b","c"])             # ["a","b","c"]
flatten([[1,2],[3,4]])               # [1,2,3,4]
distinct(["a","b","a"])              # ["a","b"]
merge({a=1}, {b=2})                  # {a=1, b=2}
lookup({a="x"}, "a", "default")     # "x"
keys({a=1, b=2})                     # ["a","b"]
values({a=1, b=2})                   # [1,2]
```

#### Numeric Functions

```hcl
max(1, 2, 3)                         # 3
min(1, 2, 3)                         # 1
ceil(1.2)                            # 2
floor(1.9)                           # 1
```

#### Type Conversion

```hcl
tostring(42)                         # "42"
tonumber("42")                       # 42
tolist(toset(["a","b"]))             # ["a","b"]
```

#### Encoding

```hcl
base64encode("hello")                # "aGVsbG8="
jsonencode({key="value"})            # "{\"key\":\"value\"}"
```

## Backends

### Common Backend Types

| Backend    | Use Case                                  |
|------------|-------------------------------------------|
| `s3`       | AWS — most popular for teams              |
| `azurerm`  | Azure Blob Storage                        |
| `gcs`      | Google Cloud Storage                      |
| `remote`   | Terraform Cloud / HCP Terraform           |
| `http`     | Custom HTTP endpoint                      |
| `consul`   | HashiCorp Consul                          |
| `local`    | Default — suitable for solo/dev only      |

> **Interview Tip:** When using `terraform init` with a backend change, Terraform will prompt to migrate existing state. Use `terraform init -migrate-state` to do this automatically.

## CLI Commands

### Initialization & Setup

```bash
terraform init                      # Initialize working directory
terraform init -upgrade             # Upgrade providers to latest allowed versions
terraform init -reconfigure         # Reconfigure backend (force re-init)
terraform init -migrate-state       # Migrate statefile to new backend
```

### Formatting & Validation

```bash
terraform fmt                       # Auto-format .tf files
terraform fmt -recursive            # Format all subdirectories too
terraform fmt -check                # Verify formatting without making changes
terraform fmt -diff                 # Show differences between formatted and current files
terraform fmt -check -diff          # Combine: check and show diff
terraform validate                  # Validate configuration syntax
```

### Planning

```bash
terraform plan                                        # Show execution plan
terraform plan -out=tfplan                            # Save plan to file
terraform plan -target=aws_instance.web               # Plan only a specific resource
terraform plan -generate-config-out=ec2.tf            # Generate .tf config from existing/imported resources
```

### Applying

```bash
terraform apply                                       # Apply changes (prompts for confirmation)
terraform apply -auto-approve                         # Skip confirmation prompt
terraform apply tfplan                                # Apply a saved plan file
terraform apply -target=module.vpc                    # Apply only a specific module/resource
terraform apply -parallelism=20                       # Increase parallel execution (default: 10)
terraform apply -var-file="dev.tfvars"                # Apply with a specific variable file
```

### Destroying

```bash
terraform destroy                                     # Destroy all managed resources
terraform destroy -target=aws_s3_bucket.logs          # Destroy a specific resource
```

### State Operations

```bash
terraform state list                                  # List all resources in state
terraform state show <resource>                       # Inspect a resource's state
terraform state mv <old> <new>                        # Rename/move a resource in state
terraform state rm <resource>                         # Remove from state (keeps real resource)
terraform state pull                                  # Download and display remote state
terraform import <resource> <id>                      # Import existing infra into state
```

### Outputs & Inspection

```bash
terraform output                                      # Show all outputs
terraform output public_ip                            # Show a specific output
terraform output -json                                # Output in JSON format
terraform show                                        # Show current state or plan
terraform graph | dot -Tpng > graph.png               # Visualize dependency graph
```

### Workspaces

```bash
terraform workspace show                              # Show the current workspace name
terraform workspace list                              # List all workspaces
terraform workspace new <name>                        # Create a new workspace
terraform workspace select <name>                     # Switch to a workspace
 
```

### Resource Recreation

```bash
terraform taint aws_instance.web                      # Mark for forced recreation (deprecated)
terraform apply -replace=aws_instance.web             # Modern replacement for taint
```

### State Locking

```bash
terraform force-unlock <lock-id>                      # Release a stuck state lock (use carefully!)
```

### Variable Files

```bash
terraform apply -var-file="dev.tfvars"
terraform apply -var-file="test.tfvars"
terraform apply -var-file="prod.tfvars"
```
