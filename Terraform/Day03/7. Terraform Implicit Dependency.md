# Terraform Implicit Dependency Example

## Overview
In Terraform, dependencies between resources can be **implicit** or **explicit**. An implicit dependency occurs when Terraform automatically detects a relationship based on resource references in the code, without needing the `depends_on` argument. This is common when one resource references another (e.g., an EC2 instance using a security group ID).

While implicit dependencies work, it's a best practice to define them explicitly using `depends_on` for better readability, maintainability, and to avoid potential issues in complex configurations.

## Example: EC2 Instance with Security Group

### Code
```terraform
resource "aws_instance" "dev-server" {
  provider      = aws.mumbai
  vpc_security_group_ids = [aws_security_group.main-sg.id]
  ami           = "ami-0e12ffc2dd465f6e4"
  instance_type = "t3.small"

  tags = {
    Name = "mumbai-web-dev"
  }
}

resource "aws_security_group" "main-sg" {
  provider    = aws.mumbai
  name        = "dev-server-sg-mumbai"
  description = "Allow all inbound traffic"

  ingress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
  tags = {
    Name = "dev-server-sg-mumbai"
  }
}
```

### Explanation
- **Implicit Dependency**: The `aws_instance` resource references `aws_security_group.main-sg.id` in `vpc_security_group_ids`. Terraform knows the security group must be created before the instance, so it handles the order automatically.
- **Why Explicit is Better**: In larger codebases, implicit dependencies can be hard to spot. Using `depends_on = [aws_security_group.main-sg]` in the instance resource makes the relationship clear and ensures Terraform processes resources in the intended order.
- **Note**: This example uses a provider alias (`aws.mumbai`) for the Mumbai region. Ensure your provider is configured accordingly.

## Best Practices
- Use `depends_on` for non-obvious dependencies (e.g., when resources don't directly reference each other).
- Test with `terraform plan` to verify dependency resolution.
- For complex graphs, consider Terraform's dependency visualization tools.

## Running the Example
1. Run `terraform init`, `terraform plan`, and `terraform apply`.
2. Verify the instance launches with the security group attached.
