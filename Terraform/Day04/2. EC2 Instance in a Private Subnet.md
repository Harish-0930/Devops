# Connecting to an EC2 Instance in a Private Subnet

This guide explains how to connect to an EC2 instance deployed in a private subnet using Terraform, where direct internet access is not possible due to the lack of a public IP address.

## Overview
Instances in private subnets cannot be accessed directly from the internet. Common methods include using a bastion host (jump box) or AWS Systems Manager (SSM). The bastion host method is recommended for setups like the one in your Terraform configuration, which includes a public instance.

## Method 1: Using a Bastion Host (Jump Box)

### Prerequisites
- A public instance (e.g., `public_instance_01`) in a public subnet with a public IP address.
- SSH key pair used to launch the instances.
- Security group allowing SSH (port 22) from the bastion host's IP or VPC CIDR.

### Steps
1. **Locate the Public IP of the Bastion Host**:
   - In the AWS EC2 Console, find the public instance and note its **Public IPv4 address** (e.g., `13.127.45.67`).

2. **SSH into the Public Instance**:
   - Open a terminal.
   - Run: `ssh -i /path/to/your/key.pem ec2-user@<public-ip>`
     - Replace `/path/to/your/key.pem` with your private key path.
     - Use the appropriate username (e.g., `ec2-user` for Amazon Linux).

3. **SSH from the Bastion Host to the Private Instance**:
   - Once connected to the public instance, note the private IP of the private instance (e.g., `10.0.3.10`).
   - Run: `ssh -i /path/to/your/key.pem ec2-user@<private-ip>`
   - You're now connected to the private instance.

### Tips
- Copy the SSH key to the bastion host if needed: `scp -i /path/to/key.pem /path/to/key.pem ec2-user@<public-ip>:/home/ec2-user/`.
- Secure the setup by restricting security group rules.
- Check instance status, key permissions, and logs if issues arise.

## Method 2: Using AWS Systems Manager (SSM) Session Manager

### Prerequisites
- SSM agent installed on the instance.
- IAM role with `AmazonSSMManagedInstanceCore` policy attached.
- Appropriate AWS permissions for your user.

### Steps
1. In the AWS EC2 Console, select the private instance.
2. Click **Connect** > **Session Manager**.
3. Click **Connect** to start a browser-based session.

### Notes
- No SSH keys or public IPs required.
- Ensure the instance is reachable via SSM.

## Other Options
- **VPN**: Set up AWS Client VPN for direct access.
- **Direct Connect/VPC Peering**: For advanced networking.

## Troubleshooting
- **Connection Refused**: Verify instance state and security groups.
- **Key Issues**: Ensure correct key pair and permissions (`chmod 400 key.pem`).
- **Routing**: Confirm private subnet routes outbound via NAT Gateway.
